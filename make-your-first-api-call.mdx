---
title: "Make your first api call"
---

In under five minutes, you'll register your institution, issue your first payment handle, execute a transfer, and start receiving real-time events through webhooks.

## 1. Register your institution

Before using the API, your institution must be registered on Zelkea.

- Go to [https://app.zelkea.com](https://app.zelkea.com) and create an institution account.
- Complete onboarding and grab your first API key from your dashboard.

During onboarding, you'll choose a domain tied to your institution. We recommend using your company name. For example, if your company is Lion Bank, choose `@lionbank`. This keeps handles memorable, easy to type, and less prone to mistakes.

## 2. Register a handle

Every customer on the network needs a unique payment handle like `tom@lionbank` to send and receive money. The domain is automatically appended based on your institution configuration.

<CodeGroup>

```bash cURL Request
curl --request POST 'https://api.zelkea.com/handles' \
--header 'Authorization: Bearer <API key>' \
--header 'Content-Type: application/json' \
--data-raw '{
  "handle": "alice", 
  "balance": 1000,
  "limit": 5000,
  "currency": "USD",
  "code": "XSO2B9",
  "status": "active"
}'
```

```json Response
{
  "handle": "alice@lionbank",
  "balance": 1000,
  "limit": 5000,
  "currency": "USD",
  "code": "XSO2B9",
  "status": "active",
  "created_at": "2026-07-10T00:21:49Z"
}
```

</CodeGroup>

## 3. Create a transfer

Use the Transfer API to initiate an instant fund transfer to any active handle in the network. Include a unique UUID in the `idempotency-key` header to prevent duplicate processing.

<CodeGroup>

```bash cURL Request
curl --request POST 'https://api.zelkea.com/transfers' \
--header 'Authorization: Bearer <API key>' \
--header 'idempotency-key: 7b9e4a3c-2d1e-4f5a-8b6c-9d8e7f6a5b4c' \
--header 'Content-Type: application/json' \
--data-raw '{
  "sender": "alice@lionbank",
  "receiver": "bob@scbank",
  "amount": 250,
  "currency": "USD"
}'
```

```json Response
{
    "id": "tx_987654321",
    "sender": "alice@lionbank",
    "receiver": "bob@scbank",
    "amount": 250,
    "currency": "USD",
    "status": "completed",
    "created_at": "2026-07-10T00:22:00Z"
  }
```

</CodeGroup>

## **4. Set up webhooks**

Now you can register a webhook endpoint to receive real-time payment and transfer events.

<CodeGroup>

```bash cURL Request
curl --request POST 'https://api.zelkea.com/webhooks' \
--header 'Authorization: Bearer <API key>' \
--header 'Content-Type: application/json' \
--data-raw '{
  "url": "https://domain.com/webhooks"
}'
```

```json Response
{
    "url": "https://domain.com/webhooks",
    "secret": "whsec_ee4..."
  }
```

</CodeGroup>

That's it. Your institution can now issue payment handles, make transfers, and receive real-time transaction events.
