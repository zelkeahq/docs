title: "Payments"

A payment is how a business collects money from a customer. It looks similar to a transfer, but with one key difference: it requires the customer's explicit, single-use authorization. This provides the same trust boundary as a card CVV or 3D Secure confirmation, without requiring either.

<Note>
  Payments are for **businesses** collecting money online. If you're an institution moving money between two handles directly, you want [Transfers](/transfers) instead.
</Note>

## **Payment authorization**

Every handle has an associated code, a short-lived rotating value retrieved on demand by the customer's institution and displayed inside their banking or wallet app. During online checkout, the customer enters the code along with their handle. The business includes it in the payment request, and the code is invalidated immediately after use.

<Tip>
  Think of it like a card's CVV, except it rotates automatically and can only be used once. Even if a code is exposed, it has no value after use and cannot be reused for another payment.
</Tip>

## Creating a payment

```bash Execute a payment
curl --request POST 'https://api.zelkea.com/payments' \
  --header 'Authorization: Bearer <API key>' \
  --header 'idempotency-key: 4f5a8b6c-9d8e-7f6a-5b4c-7b9e4a3c2d1e' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "customer": "malick@wallet",
    "amount": 14.99,
    "currency": "USD",
    "code": "7X9K2M"
  }'
```

```json Response
{
  "id": "pmt_a1b2c3d4",
  "merchant": "company@us",
  "customer": "malick@wallet",
  "amount": 14.99,
  "currency": "USD",
  "status": "completed",
  "created_at": "2026-07-10T00:22:00Z"
}
```

If the code is wrong, already used, or expired, the request is automatically rejected.

## Split payments

A single customer payment can be split across multiple recipients in a single request, making it ideal for marketplaces, revenue-sharing platforms, and any checkout flow where funds need to be distributed between multiple parties.

```json Marketplace split
{
  "customer": "malick@wallet",
  "amount": 100,
  "currency": "USD",
  "code": "7X9K2M",
  "splits": [
    { "handle": "vendor1@za", "amount": 70 },
    { "handle": "platform@dz", "amount": 30 }
  ]
}
```

The split amounts must add up to the total payment amount, otherwise the request is rejected. Each recipient receives their share directly and instantly, with no one needing to wait for another party to process a payout.

## Retrieving payments

Successful payments settle instantly. Funds are available immediately, so you do not need to rely on webhooks to know when a payment completes. If you prefer event-driven updates, you can optionally [register a webhook endpoint](/webhooks) to receive payment events in real time.

```bash List payments received
curl --request GET 'https://api.zelkea.com/payments' \
  --header 'Authorization: Bearer <API key>'
```

```bash Get one specific payment
curl --request GET 'https://api.zelkea.com/payments/pmt_a1b2c3d4' \
  --header 'Authorization: Bearer <API key>'
```
