A transfer is the core operation on Zelkea: funds move from one handle's institution balance to another, regardless of where either handle is issued.

<Note>
  Transfers are used by institutions to move money on behalf of the handles they issued. If you're a business trying to get paid by a customer, you want [Payments](/payments) instead.
</Note>

## When to use a transfer

A transfer works like a bank or mobile wallet transfer. Institutions use the Transfer API to move funds on behalf of their customers. Transfers are always instant, regardless of whether the handles belong to individuals or corporates, whether the parties are in the same country or different countries, or whether the currencies are the same or different.

## Making a transfer

Every transfer needs a unique `idempotency-key `provided by you as a UUID. If a request times out and you retry it with the same key, Zelkea returns the original result instead of processing the transfer again.

```bash Send a transfer
curl --request POST 'https://api.zelkea.com/transfers' \
  --header 'Authorization: Bearer <API key>' \
  --header 'idempotency-key: 7b9e4a3c-2d1e-4f5a-8b6c-9d8e7f6a5b4c' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "sender": "alice@acme",
    "receiver": "bob@scbank",
    "amount": 250,
    "currency": "USD",
    "reference": "invoice-4471"
  }'
```

```json Response
{
  "id": "tx_987654321",
  "sender": "alice@acme",
  "receiver": "bob@scbank",
  "amount": 250,
  "currency": "USD",
  "status": "completed",
  "created_at": "2026-07-10T00:22:00Z",
  "reference": "invoice-4471"
}
```

`reference` is optional but recommended. You can pass your own internal ID, invoice number, or ticket reference, and Zelkea returns it unchanged whenever you retrieve the transfer. This lets you link Zelkea transactions back to your own records without maintaining a separate mapping table.

<Warning>
  Transfers settle instantly and are **final**. There is no reversal endpoint because Zelkea only routes money between institutions. If money needs to be returned, a new transfer must be initiated in the opposite direction, just like correcting a traditional bank transfer.
</Warning>

## Looking a transfer up later

Every transfer you send can be retrieved again, either as a list or by ID:

```bash Get transfer history for a handle
curl --request GET 'https://api.zelkea.com/transfers/alice@acme' \
  --header 'Authorization: Bearer <API key>'
```

```bash Get one specific transfer
curl --request GET 'https://api.zelkea.com/transfers/alice@acme/tx_987654321' \
  --header 'Authorization: Bearer <API key>'
```

## Finding out the instant it happens

If your institutional balance is suffisant, outbound transfers are instant. For inbound transfers, we recommend you register [register a webhook endpoint](/webhooks) to receive real time transaction updates.
