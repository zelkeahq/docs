## Why generated, not hand-written

Our SDKs are generated directly from our [OpenAPI specification](https://docs.zelkea.com/openapi.json), rather than maintained by hand. This means every SDK stays in lockstep with the API itself — a new field or endpoint in the spec shows up in the SDK automatically, with no drift between what's documented and what the client library actually supports.

## In the meantime

The spec itself is open and can be fed into any standard OpenAPI code generator if you'd like typed bindings before an official SDK for your language ships:

```bash
npx @openapitools/openapi-generator-cli generate \
  -i https://docs.zelkea.com/openapi.json \
  -g typescript-fetch \
  -o ./zelkea-client
```

This isn't the official SDK and isn't supported directly by us, but since it's generated from the same spec we publish, it'll track the real API accurately.
